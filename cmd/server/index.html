Display Preferences

These preferences are stored in your browser only. They do not affect the server.

The server renders white-on-black matching the real amp display. “Inverted” is the default. “LCD Native” flips the display image in your browser only via CSS filter. It does not affect the server or any API consumers.

Connection

Saved to the server config file. Serial port, listen address, and display-frame refresh interval require a server restart to take effect. Display polling and status polling are live toggles.

The picker enumerates ports the server can see. Select one to fill the field, then save.

Backend display-frame polling cadence. This affects how often the server asks the amp for display/frame data. The web UI mostly reads cached state, so refreshing the page faster will not outrun this cadence.

Live toggle — pauses or resumes backend display-frame polling immediately without a restart.

Live toggle — pauses or resumes backend protocol-status refresh immediately without a restart. /api/v1/status and the websocket status feed mostly serve cached in-memory state built from this path.

Restart asks this process to exit cleanly. It only comes back automatically when Expert Amp Server is being run under a supervisor such as systemd, launchd, Docker restart policy, or a similar service wrapper.

Advanced Serial

Advanced serial connection parameters. Saved to server config; take effect after restart except where noted. Display-frame refresh and protocol-status refresh are separate backend paths so you can run either path or both. Defaults match the SPE Expert serial interface.

When enabled, the server sends the documented status poll command from the Programmer's Guide to refresh protocol-native status in memory. The command itself is fixed in code so it cannot be accidentally broken in settings.

Backend protocol-status cadence. Websocket clients and most API consumers read cached state from this path, so polling the API faster than this mainly just reads the same snapshot more often.
